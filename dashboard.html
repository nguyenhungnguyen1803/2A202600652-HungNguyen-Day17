Tóm Tắt Bài Tập Hôm Nay

Bài tập hôm nay yêu cầu cân bằng giữa khả năng ghi nhớ dài hạn (Recall) và chi phí vận hành (Token Usage/Context Window) của AI Agent.

Ba lớp Memory đã xây dựng:

1. Short-term Memory: Lưu trữ trong thread hiện tại (Baseline và Advanced).

2. Persistent Memory: Đọc ghi file markdown cấu trúc User.md để nhớ facts bền vững qua các thread mới.

3. Compact Memory: Nén (Summarize) tin nhắn cũ khi vượt ngưỡng để tối ưu hóa context size.

Các Tính Năng Nâng Cao Đã Có (90-100 Điểm)

  • ✓ Conflict Handling: Tự động ghi đè, cập nhật thông tin đính chính mới nhất của người dùng (ví dụ: đổi nơi ở từ Huế sang Đà Nẵng) để tránh xung đột dữ liệu.
  • ✓ Structured Entity Extraction: Phân tách facts và ghi lưu trữ thành các cặp Khóa: Giá trị có cấu trúc rõ ràng trong markdown để dễ dàng phân tích ngoại tuyến.
  • ✓ Question Filtering: Bỏ qua các tin nhắn mang tính chất đặt câu hỏi thuần túy, tránh lưu nhầm câu hỏi thành facts của user.

model_provider.pydef normalize_provider(value: str) -> str:
    val = value.lower().strip()
    if val in ["google", "gemini"]: return "gemini"
    return val

def build_chat_model(config: ProviderConfig):
    provider = normalize_provider(config.provider)
    if provider == "openai":
        from langchain_openai import ChatOpenAI
        return ChatOpenAI(model=config.model_name, temperature=config.temperature, api_key=config.api_key)

Lớp Lưu Trữ File markdown & Compact Memory

Quản lý file User.md và cơ chế nén nén bộ nhớ (Compaction). 

memory_store.pyclass UserProfileStore:
    def read_facts(self, user_id: str) -> dict[str, str]:
        content = self.read_text(user_id)
        facts = {}
        for line in content.splitlines():
            if line.startswith("- ") and ":" in line:
                k, v = line[2:].split(":", 1)
                facts[k.strip().lower()] = v.strip()
        return facts

Baseline Agent (Chỉ nhớ trong Thread)

agent_baseline.pyclass BaselineAgent:
    def _reply_offline(self, thread_id: str, message: str) -> dict[str, Any]:
        if thread_id not in self.sessions:
            self.sessions[thread_id] = SessionState()
        session = self.sessions[thread_id]
        
        prompt_tokens = sum(estimate_tokens(m["content"]) for m in session.messages)
        session.prompt_tokens_processed += prompt_tokens
        session.messages.append({"role": "user", "content": message})
        
        ans = "Tôi là Baseline Agent và không nhớ thông tin cũ ngoài thread này."
        reply_tokens = estimate_tokens(ans)
        session.token_usage += reply_tokens
        session.messages.append({"role": "assistant", "content": ans})
        return {"response": ans, "tokens": reply_tokens, "prompt_tokens": prompt_tokens}

Advanced Agent (3 Tầng Memory)

agent_advanced.pyclass AdvancedAgent:
    def _reply_offline(self, user_id: str, thread_id: str, message: str) -> dict[str, Any]:
        facts = self.profile_store.read_facts(user_id)
        new_facts = extract_profile_updates(message)
        facts.update(new_facts)
        self.profile_store.write_facts(user_id, facts)
        
        self.compact_memory.append(thread_id, "user", message)
        prompt_tokens = self._estimate_prompt_context_tokens(user_id, thread_id)
        
        response_text = self._offline_response(user_id, thread_id, message)
        self.compact_memory.append(thread_id, "assistant", response_text)
        reply_tokens = estimate_tokens(response_text)
        
        self.thread_tokens[thread_id] = self.thread_tokens.get(thread_id, 0) + reply_tokens
        self.thread_prompt_tokens[thread_id] = self.thread_prompt_tokens.get(thread_id, 0) + prompt_tokens
        return {"response": response_text, "tokens": reply_tokens, "prompt_tokens": prompt_tokens}

Luồng Xử Lý Của Advanced Memory System

Sơ đồ dưới đây minh họa cách Advanced Agent tiếp nhận một tin nhắn của người dùng, phân phối vào các tầng nhớ và tối ưu ngữ cảnh.

1
User Message Received
Người dùng gửi tin nhắn vào hệ thống kèm theo user_id và thread_id.
2
Fact Extraction & Profile Sync
Agent trích xuất thông tin mới (nơi ở, nghề nghiệp...) và cập nhật ngay lập tức vào file markdown User.md.
3
Short-Term Memory Append
Thêm tin nhắn mới vào CompactMemoryManager của thread hiện tại.
4
Compaction Check (Nén bộ nhớ)
Nếu tổng số token của thread vượt ngưỡng, nén các tin nhắn cũ hơn thành 1 dòng summary chung và giải phóng các tin nhắn cũ.
5
Context Window Assembly
Lắp ráp Prompt gồm: User.md + Summary + Recent Messages (mới nhất) đưa vào LLM sinh câu trả lời.
